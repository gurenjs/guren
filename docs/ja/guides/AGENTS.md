# ドキュメント更新ガイドライン（日本語要約）

このリポジトリのドキュメントを編集する際の基本方針です。詳しくは英語版 [AGENTS.md](../../guides/AGENTS.md) を確認してください。

## 前提
- 読者は `create-guren-app` で新規プロジェクトを作る開発者。
- フレームワーク内部のパッケージ構成や monorepo の詳細には踏み込まない。

## 書き方の原則
- ステップバイステップで、プロジェクト生成 → 依存インストール → 環境変数 → DB 準備 → dev サーバー起動、の流れを重視。
- 例はフレームワークに同梱される機能のみを扱う。内部ツール名（`citty` など）は出さない。

## トーン
- 簡潔でフレンドリー。能動態で「あなた」を主体に。
- コマンドはフェンス付きコードブロックで、何を達成するかを短く説明。

## リンクと整合性
- 相互リンクは相対パスで。入口は `overview.md`。
- Quick Start と Getting Started の内容がずれたら両方更新する。
- 禁止用語や outdated な例がないか `docs/` で `rg` を走らせて確認。

この方針に沿って翻訳や追記を行うと、ドキュメントの一貫性を保てます。
